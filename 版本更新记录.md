# 软件工程智能体 —— 版本更新记录

| 编号 | 日期 | 版本 | 修改内容概述 |
|:----:|------|:----:|------|
| 01 | 2026-05-05 | v0.3 | **项目初始化与核心功能实现**。创建项目结构，完成数据模型定义（models.py）、LLM 适配器（支持 Anthropic/OpenAI）、需求分析器（单次 LLM 调用提取实体/行为/约束）、图表生成器（类图/活动图/状态机图，Mermaid 格式）、任务编排器（串行流程）、输出格式化器和 CLI 入口 |
| 02 | 2026-05-05 | v0.4 | **接入 DeepSeek 模型**。LLM 适配器新增 DeepSeek 支持（OpenAI 兼容接口），默认模型切换为 DeepSeek V4 Pro，新增 `api_base`/`api_key` 配置字段，自动读取环境变量 |
| 03 | 2026-05-06 | v0.8 | **并发架构改造**。分析器拆为骨架提取+并发细节填充（ThreadPoolExecutor），图表生成阶段全部并发执行；新增进度提示（始终输出无需 --verbose） |
| 04 | 2026-05-06 | v0.10 | **架构回退与稳定**。分析器回退为单次 LLM 调用（并发拆分在推理模型下更慢），图表阶段保持并发；移除 PNG 导出和流式输出，精简代码 |
| 05 | 2026-05-07 | v0.15 | **鲁棒性修复与配置完善**。修复 LLM 返回参数格式不一致导致的崩溃（`_normalize_string_list` 统一转换）；修复 `from_yaml` 中 `field()` 默认值问题；默认模型切换为 DeepSeek V4 Flash；并发上限从 10 降为 3 避免限速 |
| 06 | 2026-05-07 | v0.18 | **透明审核 + IDE 集成**。新增审核编辑器（AI 提取后暂停展示，支持人工增删改实体/行为）；新增 `--skip-review`、`--edit-file` 参数；数据模型增加序列化方法；配置 VSCode Tasks（Ctrl+Shift+B 快捷启动）；缓存优化（system prompt 恒定为固定模板） |
| 07 | 2026-06-04 | v0.20 | **文档与基准测试完善**。补齐 5 份跨领域基准测试 PRD（图书管理、在线考试、停车管理、餐饮点餐、医院挂号）；编写最终汇报文档（含架构设计、核心技术方案、基准测试结果）；生成版本更新记录 |
| 08 | 2026-06-07 | v0.21 | **分析 Prompt 优化与遗漏检测**。分析 Prompt 增加辅助流程提取规则（定时任务/消息推送/提醒通知/日志记录/数据备份），减少支线逻辑遗漏；新增关键词扫描机制——分析完成后自动检测 PRD 中"通知/提醒/日志/备份/续借/注册/取消/统计/导出"等关键词是否被 AI 覆盖，未覆盖则提示用户审核时注意；审核预览增加属性和方法详情展示（组员D） |
| 09 | 2026-06-07 | v0.22 | **泛用 Prompt 重构与通用遗漏检测**（组员B）。分析 Prompt 彻底重构为泛用规则——去掉全部硬编码领域关键词，改为 6 条通用提取规则（每个功能子章节→独立行为/实体、系统自动触发流程与用户触发同等重要、多服务模式→独立流程、统计分析也是独立行为）；遗漏检测从固定 15 关键词改为自动从 PRD 提取功能项并匹配 60+ 通用业务动词；图书管理系统行为覆盖率 50%→100%（3→18个行为）、实体覆盖率 86%→100%（6→9个实体）；餐饮系统行为覆盖率 73%→89%（外卖/预约/购物车/统计从无到有） |
